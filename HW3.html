In this assignment you will first implement a Hash Table with Buckets, and then use it for implementing the Concordance function. Let us break each of those tasks down step-by-step.

1. Hash Table

A hash table (or map) is a data structure that stores elements defined as (key,value) pairs. The interface of a hash table includes the following functions:

   void initMap (struct hashMap * ht, int tableSize);

   int containsKey (struct hashMap * ht, KeyType k);

   void insertMap (struct hashMap * ht, KeyType k, ValueType v);

   ValueType atMap (struct hashMap * ht, KeyType k);

   void removeKey (struct hashMap * ht, KeyType k);

   int sizeMap (struct hashMap *ht); 

The function initMap() initializes the hash table with a given table size. The function containsKey() returns a non-zero integer if the key is found in the hash table, and zero, otherwise. The function insertMap() overwrites any previous value associated with the input key. The function atMap() returns a pointer to value associated with the input key, or returns NULL  if the key is not found. The function removeKey() removes an element with a given key from the hash table. The function sizeMap() returns the number of elements in the map. The skeleton code also provides two versions of hash functions for computing an integer value from a string key: 

int stringHash1(char * str);

int stringHash2(char * str);

Your task will be to complete some of the aforementioned functions and in this way complete implementation of a hash table with buckets.

2. Concordance

Once you have your Hash Table working, you will implement the concordance function using the hash table. Concordance is a function that counts the number of times each word is used in the input text file. For example, if the input text is:

   "It was the best of times,  It was the worst of times."

The output of the Concordance function will be:

   best: 1

   It: 2

   was: 2

   the: 2

   of: 2

   worst: 1

Given the input text, you will construct the hash table where keys are words found in the input text, and values are counts of how many times the corresponding words appear in the input text.

3. Main function

Your main function in main.c must accept one command-line argument as the input text-file name. This text file will then be opened to count word frequencies in the text. Your program should: a) Open the available text file *.txt, b) Read the file, c) Count the frequency of each word found in the input text file, and d) Print out the word frequencies. For the purposes of this assignment, a word may contain only small and capital letters 'a' to 'z' and 'A' to 'Z'; numbers 0 to 9; and apostrophe, but no other symbols (e.g., dash, underscore, and period are not allowed). The case is important (e.g., "It" and "it" are different words). The words found in the input text can be printed in any order (not necessarily in the order of appearance). Note that for grading this homework, we will use another, our own, input text file.

5. Homework Submission

Make sure your code compiles with our Makefile on the School's Linux CPU server. Please submit on Canvas the following two C files:

  • hashMap.c

  • main.c

 

6. Grading policy

There is no partial credit for individual functions. We will automatically grade your homework using a script code. This means that your code must compile without errors and run without any memory issues so that we can grade it. Our grading will be based on the following: 

  •  0 points = Code does not compile; or code compiles well, but has memory issues when running.
  • 25 points = Code runs, but does not read the input text correctly and gives wrong words and wrong counts for such words.
  • 35 points = Code runs, but misses to print out some words from the input text, and for some words gives a wrong count.
  • 50 points = Code runs, but misses to print out some words from the input text, and for all printed words gives the correct count.
  • 65 points = Code runs, prints out all words from the input text correctly, but for some words gives a wrong count.
  • 80 points = Code runs, prints out all words from the input text, but gives some wrong word counts
  • 100 points = Code runs, prints out all words from the input text, and gives all correct word counts

 

7. Compiling errors of your code

For grading, we will consider ONLY the latest submission of your homework. 

    • Verify whether your code compiles without errors using our Makefile on the Schools server.
    • Within a few hours after the submission deadline, but no later than 2pm, you will get an email notification from a TA whether your source code compiles with errors. In that case, you will need to correct and resubmit your code by 5pm of the same day. You will be given only one chance to correct your compiling errors. Importantly, we will not be able to notify you about compiling errors if you submitted your homework late.
    • Uncorrected code with compiling errors will get zero credit.
